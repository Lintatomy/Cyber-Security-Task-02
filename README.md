Part A: Understanding the CIA Triad :-

1. Confidentiality Definition

Confidentiality means ensuring that information is accessible only to authorized users. It prevents unauthorized individuals from viewing sensitive data.

-> Why It Is Important
Protects personal and organizational information.
Prevents identity theft and data leakage.
Maintains customer trust.
Helps organizations comply with privacy laws.

Real-World Example
A hospital stores patient medical records. Only doctors, nurses, and authorized staff can access these records. Unauthorized users are denied access.

2. Integrity Definition

Integrity means ensuring that data remains accurate, complete, and unaltered unless modified by authorized users.

-> Why It Is Important
Prevents unauthorized modification of data.
Ensures reliable business operations.
Maintains trust in digital systems.
Reduces the risk of incorrect information.

Real-World Example
A university stores students' examination results. Only authorized faculty members can update grades, ensuring students receive accurate marks.

3. Availability Definition
4. 
Availability means ensuring that systems, applications, and data are accessible whenever authorized users need them.

-> Why It Is Important
Prevents service interruptions.
Supports continuous business operations.
Improves customer satisfaction.
Reduces downtime.

Real-World Example
An online banking website must remain available 24/7 so customers can access their accounts anytime.

Part B: Real-World Case Study :-

WannaCry Ransomware Attack (2017)

-> What Happened?
WannaCry was a ransomware attack that spread rapidly across more than 150 countries. It exploited a vulnerability in Microsoft Windows systems and encrypted users' files. Victims had to pay a ransom in Bitcoin to regain access to their files.

-> Which Part of the CIA Triad Was Affected?
--Availability
The ransomware encrypted files, preventing users from accessing their data.
--It also affected:
Integrity (files were altered through encryption)
Confidentiality (attackers gained unauthorized control over systems)

--Impact
*More than 200,000 computers infected.
*Hospitals cancelled surgeries.
*Businesses experienced major downtime.
*Millions of dollars in financial losses.
*Government organizations were affected.

--Prevention

*Install security patches regularly.
*Keep operating systems updated.
*Use antivirus software.
*Maintain regular backups.
*Train employees to recognize suspicious emails.
*Disable outdated protocols such as SMBv1.

Part D: Threat Identification :-

1. Insider Threat
   Description
An insider threat occurs when an employee, student, or authorized user intentionally or accidentally compromises security.

Possible Impact--

* Data theft
* Information leakage
* Financial loss
* Reputation damage

Prevention Methods-- 
* Role-based access control
* User activity monitoring
* Employee awareness training
* Least privilege principle

2. Malware
   Description
Malware is malicious software designed to damage, steal, or disrupt computer systems.

Examples include--
* Virus
* Worm
* Trojan
* Spyware
* Ransomware

Possible Impact--
* Data loss
* Slow computer performance
* Financial damage
* Unauthorized access

Prevention Methods--
* Antivirus software
* Software updates
* Email filtering
* Safe browsing practices

3. Phishing
   Description
Phishing is a cyber attack where attackers trick users into revealing passwords, banking information, or personal details using fake emails or websites.

Possible Impact--
* Account compromise
* Identity theft
* Financial fraud
* Data breach
  
Prevention Methods--
* Employee awareness
* Verify email addresses
* Enable MFA
* Avoid suspicious links

4. Weak Passwords
   Description
Weak passwords are easy to guess or crack using brute-force attacks.

Possible Impact--
* Unauthorized account access
* Data theft
* System compromise

Prevention Methods--
* Strong passwords
* Password manager
* MFA
* Regular password changes

5. Unpatched Systems
   Description
Unpatched systems have outdated software containing known security vulnerabilities.

Possible Impact--
* Malware infection
* Hacker access
* Data breach
* System downtime
  
Prevention Methods--
* Automatic updates
* Patch management
* Vulnerability scanning
* Regular maintenance

Part E: Security Recommendations :-

1. Enable Multi-Factor Authentication (MFA)
   Importance:
Adds an extra layer of security beyond passwords.

2. Regular System Updates
   Importance:
Fixes known vulnerabilities and prevents cyber attacks.

3. Strong Password Policy
   Importance:
Makes passwords harder to guess or crack.

4. Employee Security Training
   Importance:
Helps users recognize phishing emails and cyber threats.

5. Regular Data Backups
   Importance:
Allows recovery after ransomware attacks or hardware failure.

6. Install Antivirus Software
   Importance:
Detects and removes malicious software.

7. Firewall Protection
   Importance:
Blocks unauthorized network access.

8. Access Control
   Importance:
Ensures only authorized users access sensitive data.

9. Encrypt Sensitive Data
   Importance:
Protects confidential information even if stolen.

10. Continuous Security Monitoring
    Importance:
Detects suspicious activities early and reduces attack impact.

     Cybersecurity Improvement Report for College
     
  Introduction
Cybersecurity is essential for protecting student information, academic records, and digital infrastructure. As educational institutions increasingly rely on technology, implementing strong security measures is necessary to safeguard sensitive data and ensure uninterrupted operations.

--Top 5 Risks--
* Student data breaches
* Phishing attacks targeting faculty and students
* Malware and ransomware infections
* Weak passwords leading to unauthorized access
* Unpatched software vulnerabilities

--Top 5 Recommendations --
* Implement Multi-Factor Authentication (MFA)
* Conduct regular software updates and patch management
* Provide cybersecurity awareness training
* Maintain regular backups of critical data
* Enforce strong password policies

--Expected Benefits --
* Better protection of student and faculty information
* Reduced risk of cyber attacks
* Improved system reliability and availability
* Faster recovery from incidents
* Increased trust among students, staff, and stakeholders

--Conclusion ---
By adopting these cybersecurity practices, the college can significantly reduce security risks, protect sensitive information, and maintain reliable digital services. Continuous monitoring, employee awareness, and proactive security measures are key to building a strong cybersecurity posture.
